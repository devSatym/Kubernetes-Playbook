Here are **battle-tested `command:` / `args:` shell snippets** that are genuinely useful inside Kubernetes Pods — especially for **initContainers, debugging, dependency waiting, health simulation, startup orchestration, sidecars, and exam scenarios**.

---

# Core Pattern

```yaml
command: ["sh", "-c", "<script>"]
```

---

# 1. Wait for DNS / Service Discovery

### Wait until service resolves

```yaml
command: ["sh", "-c", "until nslookup mydb; do echo waiting for dns; sleep 2; done"]
```

### Wait until multiple services resolve

```yaml
command: ["sh", "-c", "until nslookup redis && nslookup postgres; do sleep 2; done"]
```

---

# 2. Wait for TCP Port Open

### DB readiness

```yaml
command: ["sh", "-c", "until nc -z mydb 5432; do echo waiting for db; sleep 2; done"]
```

### HTTP service readiness

```yaml
command: ["sh", "-c", "until wget -qO- http://backend:8080/health; do sleep 3; done"]
```

---

# 3. Delay Startup

### Basic sleep

```yaml
command: ["sh", "-c", "sleep 30"]
```

### Infinite sleep (debug pod)

```yaml
command: ["sh", "-c", "sleep infinity"]
```

### Keep pod alive

```yaml
command: ["sh", "-c", "tail -f /dev/null"]
```

---

# 4. Retry Logic

### Retry command 10 times

```yaml
command: ["sh", "-c", "for i in $(seq 1 10); do curl backend:8080 && exit 0; sleep 2; done; exit 1"]
```

---

# 5. Config / Secret Validation

### Wait for mounted file

```yaml
command: ["sh", "-c", "until [ -f /etc/config/app.conf ]; do sleep 2; done"]
```

### Ensure env var exists

```yaml
command: ["sh", "-c", "[ -z \"$DB_HOST\" ] && echo missing DB_HOST && exit 1 || echo ok"]
```

---

# 6. Generate Dynamic Config

### Write env into config

```yaml
command: ["sh", "-c", "echo DB_HOST=$DB_HOST > /config/app.env && exec myapp"]
```

### Template replace

```yaml
command: ["sh", "-c", "sed 's/{{DB_HOST}}/'\"$DB_HOST\"'/g' /template > /config/app.conf && exec myapp"]
```

---

# 7. Data Seeding / Init Tasks

### Seed DB once

```yaml
command: ["sh", "-c", "until nc -z mysql 3306; do sleep 2; done; mysql -h mysql < seed.sql"]
```

---

# 8. Logging Loops

### Repeated timestamp

```yaml
command: ["sh", "-c", "while true; do date; sleep 5; done"]
```

### Simulate app logs

```yaml
command: ["sh", "-c", "i=0; while true; do echo log-$i; i=$((i+1)); sleep 2; done"]
```

---

# 9. Crash Simulation

### Fail intentionally

```yaml
command: ["sh", "-c", "echo crashing; exit 1"]
```

### Crash after delay

```yaml
command: ["sh", "-c", "sleep 20; exit 1"]
```

---

# 10. File Watchers

### Wait for file then execute

```yaml
command: ["sh", "-c", "until [ -f /shared/ready ]; do sleep 1; done; exec app"]
```

---

# 11. Sidecar Sync

### Producer

```yaml
command: ["sh", "-c", "echo ready > /shared/status; tail -f /dev/null"]
```

### Consumer

```yaml
command: ["sh", "-c", "until grep -q ready /shared/status; do sleep 1; done; exec app"]
```

---

# 12. Backup / Copy

### Copy data before start

```yaml
command: ["sh", "-c", "cp -r /seed/* /data/ && exec app"]
```

---

# 13. Permission Fixes

### Chown volume

```yaml
command: ["sh", "-c", "chown -R 1000:1000 /data && exec app"]
```

---

# 14. Node / Metadata Debugging

### Show pod info

```yaml
command: ["sh", "-c", "echo Pod: $(hostname); env; sleep infinity"]
```

---

# 15. Kubernetes API Wait (ServiceAccount)

### Wait for token mount

```yaml
command: ["sh", "-c", "until [ -f /var/run/secrets/kubernetes.io/serviceaccount/token ]; do sleep 2; done"]
```

---

# 16. Parallel Dependency Wait

```yaml
command: ["sh", "-c", "until nc -z db 5432 && nc -z redis 6379; do sleep 2; done; exec app"]
```

---

# 17. Leader-like Lock File

```yaml
command: ["sh", "-c", "if mkdir /tmp/lockdir; then echo leader; else echo follower; fi; sleep infinity"]
```

---

# 18. Continuous HTTP Server (busybox)

```yaml
command: ["sh", "-c", "while true; do echo -e 'HTTP/1.1 200 OK\n\nhello' | nc -l -p 8080; done"]
```

---

# 19. Conditional Branching

```yaml
command: ["sh", "-c", "if [ \"$ENV\" = prod ]; then exec prod-app; else exec dev-app; fi"]
```

---

# 20. Best Practice Wrapper

### Always use `exec`

```yaml
command: ["sh", "-c", "prep-command && exec main-app"]
```

**Why:** `exec` replaces shell with app → proper signal handling (SIGTERM).

```yaml

- sh
- -c
- |
  until nslookup postgres; do
    echo waiting postgres
    sleep 2
  done
  until nc -z postgres 5432; do
    echo waiting port
    sleep 2
  done
```

# Most Important `command` / `args` Shell Patterns for CKA (High-Yield)

These are the **real exam-useful snippets** — the ones most likely to save time in troubleshooting, initContainers, sidecars, probes, DNS/service checks, and “make pod wait until X”.

---

# 1. Wait for Service DNS (VERY COMMON)

```yaml
command: ["sh", "-c", "until nslookup myservice; do sleep 2; done"]
```

### Use When:

* Pod must wait for Service creation
* InitContainer dependency
* DNS troubleshooting

---

# 2. Wait for Port (DB / App readiness)

```yaml
command: ["sh", "-c", "until nc -z mydb 5432; do sleep 2; done"]
```

### Use When:

* Wait for MySQL/Postgres
* Backend readiness
* TCP checks

---

# 3. Keep Debug Pod Running

```yaml
command: ["sh", "-c", "sleep infinity"]
```

### Alternative:

```yaml
command: ["sh", "-c", "tail -f /dev/null"]
```

### Use When:

* Temporary debug pod
* `kubectl exec`
* Volume inspection

---

# 4. Delay Start

```yaml
command: ["sh", "-c", "sleep 30"]
```

### Use When:

* Simulate startup delay
* Test readiness/liveness
* Sequencing

---

# 5. Run Multiple Commands (SUPER IMPORTANT)

```yaml
command:
- sh
- -c
- |
  until nslookup db; do sleep 2; done
  until nc -z db 5432; do sleep 2; done
  exec app
```

### Use When:

* InitContainers
* Complex startup chain
* Common exam editing task

---

# 6. Crash Pod Intentionally

```yaml
command: ["sh", "-c", "exit 1"]
```

### Use When:

* RestartPolicy testing
* CrashLoopBackOff demo

---

# 7. Print Logs Continuously

```yaml
command: ["sh", "-c", "while true; do date; sleep 5; done"]
```

### Use When:

* Log validation
* Sidecar logging
* `kubectl logs`

---

# 8. Wait for File / Volume

```yaml
command: ["sh", "-c", "until [ -f /data/ready ]; do sleep 2; done"]
```

### Use When:

* Shared volume sync
* Sidecar coordination
* Config generation

---

# 9. Environment Variable Validation

```yaml
command: ["sh", "-c", "[ -z \"$DB_HOST\" ] && exit 1 || sleep infinity"]
```

### Use When:

* ConfigMap/Secret debugging
* Env validation

---

# 10. Busybox HTTP Simulation

```yaml
command: ["sh", "-c", "echo hello > index.html && httpd -f -p 8080"]
```

### Use When:

* Fast test server
* Service troubleshooting
* Port-forward validation

# Exam Strategy

### If task says:

**“Wait until service is available”**
→ Use:

```yaml
until nslookup svc; do sleep 2; done
```

### If:

**“Wait until DB is reachable”**
→ Use:

```yaml
until nc -z db 5432; do sleep 2; done
```

### If:

**“Keep pod running for inspection”**
→ Use:

```yaml
sleep infinity
```

## DNS → `nslookup`

## Port → `nc -z`

## Hold → `sleep infinity`

## Multi-step → `sh -c`

## Proper startup → `exec app`
