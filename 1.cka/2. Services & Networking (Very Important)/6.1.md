If your goal is **CKA/CKS-level Kubernetes DNS mastery**, don't just memorize the CoreDNS ConfigMap. Learn **what each plugin does, when to modify it, and what problems it solves**.

The CoreDNS configuration is usually stored in:

```bash
kubectl -n kube-system get configmap coredns -o yaml
```

Typical Corefile:

```txt
.:53 {
    errors
    health
    ready
    kubernetes cluster.local in-addr.arpa ip6.arpa {
       pods insecure
       fallthrough in-addr.arpa ip6.arpa
       ttl 30
    }
    prometheus :9153
    forward . /etc/resolv.conf
    cache 30
    loop
    reload
    loadbalance
}
```

---

# 1. Change DNS Cache Time

Current:

```txt
cache 30
```

Increase cache:

```txt
cache 300
```

Effect:

* Reduces DNS queries
* Faster responses
* Records remain cached longer

Use when:

* Large clusters
* Heavy DNS traffic

---

# 2. Change DNS Record TTL

Current:

```txt
ttl 30
```

Increase:

```txt
ttl 300
```

Effect:

```txt
pod -> dns record cached for 5 min
```

Useful when:

* Services don't change frequently

---

# 3. Add Custom DNS Entries

Example:

```txt
hosts {
    10.0.0.10 db.internal
    10.0.0.20 cache.internal
    fallthrough
}
```

Now:

```bash
nslookup db.internal
```

returns:

```txt
10.0.0.10
```

Useful for:

* Legacy systems
* Lab environments

---

# 4. Forward Specific Domain to Another DNS Server

Example:

```txt
corp.local {
    forward . 10.1.1.100
}
```

Meaning:

```txt
*.corp.local
```

queries go to:

```txt
10.1.1.100
```

Useful:

* Hybrid cloud
* Corporate DNS integration

---

# 5. Forward All External Queries to Custom DNS

Current:

```txt
forward . /etc/resolv.conf
```

Change:

```txt
forward . 8.8.8.8 1.1.1.1
```

Now:

```txt
google.com
github.com
```

resolved by Google/Cloudflare.

---

# 6. Enable Query Logging

Add:

```txt
log
```

Example:

```txt
.:53 {
    log
    kubernetes cluster.local
    ...
}
```

Check:

```bash
kubectl logs -n kube-system deploy/coredns
```

Shows:

```txt
A kubernetes.default.svc.cluster.local
```

Useful for troubleshooting.

---

# 7. Enable Error Logging

```txt
errors
```

Shows:

```txt
SERVFAIL
NXDOMAIN
timeouts
```

Very useful during debugging.

---

# 8. Rewrite DNS Requests

Example:

```txt
rewrite name old.example.com new.example.com
```

Query:

```txt
old.example.com
```

becomes:

```txt
new.example.com
```

Useful:

* Migration
* DNS aliases

---

# 9. Block Domains

```txt
template IN A badsite.com {
    answer "{{ .Name }} 60 IN A 127.0.0.1"
}
```

Now:

```txt
badsite.com
```

returns:

```txt
127.0.0.1
```

Useful:

* Security
* Testing

---

# 10. Add DNS Load Balancing

```txt
loadbalance
```

Without:

```txt
10.1.1.1
10.1.1.2
10.1.1.3
```

always same order.

With:

```txt
10.1.1.2
10.1.1.1
10.1.1.3
```

rotated.

Useful for multi-endpoint services.

---

# 11. Serve Different Zones

Example:

```txt
internal.company.com:53 {
    hosts {
        10.0.0.10 app.internal.company.com
    }
}
```

CoreDNS becomes authoritative for:

```txt
*.internal.company.com
```

---

# 12. Health Endpoint

```txt
health
```

Check:

```bash
curl localhost:8080/health
```

Returns:

```txt
OK
```

Used by Kubernetes readiness/liveness probes.

---

# 13. Metrics for Prometheus

```txt
prometheus :9153
```

Access:

```bash
curl localhost:9153/metrics
```

Metrics:

```txt
coredns_dns_requests_total
coredns_cache_hits_total
```

---

# 14. DNS Failover

```txt
forward . 8.8.8.8 1.1.1.1 9.9.9.9
```

If one DNS fails:

```txt
8.8.8.8 ❌
1.1.1.1 ✅
```

CoreDNS automatically retries.

---

# 15. Stub Domain Configuration (Common Exam Topic)

Forward only:

```txt
acme.local
```

to another DNS:

```txt
acme.local {
    forward . 10.10.10.10
}
```

Everything else:

```txt
forward . /etc/resolv.conf
```

Flow:

```txt
app.acme.local
          ↓
     10.10.10.10

google.com
          ↓
     normal DNS
```

# Commands You Should Practice

```bash
kubectl -n kube-system edit configmap coredns
```

```bash
kubectl -n kube-system rollout restart deploy coredns
```

```bash
kubectl -n kube-system get pods
```

```bash
kubectl logs -n kube-system deploy/coredns
```

```bash
kubectl exec -it pod-name -- nslookup kubernetes.default
```

```bash
kubectl exec -it pod-name -- dig google.com
```

For CKA/CKS, focus especially on:

1. `forward`
2. `cache`
3. `log`
4. `errors`
5. `hosts`
6. `kubernetes`
7. Stub domains (`corp.local → custom DNS`)
8. Troubleshooting DNS resolution inside Pods
9. Reading and modifying the CoreDNS ConfigMap
10. Understanding how `forward . /etc/resolv.conf` interacts with node DNS settings

These are the CoreDNS tasks you're most likely to encounter in real clusters and certification labs.
