# 🔧 jq — JSON Processing for CKA

> **What it does:** Command-line JSON processor. Parse, filter, and transform JSON output.  
> **Why CKA cares:** `kubectl -o json` outputs JSON. jq lets you extract exactly what you need.

---

## 🧠 Core Syntax

```bash
command | jq 'filter'
jq 'filter' file.json
```

**The `.` (dot) = current object**

---

## 🔥 Essential jq Filters

### Basic Access

```bash
# Pretty-print JSON
kubectl get pod nginx -o json | jq .

# Get specific field
kubectl get pod nginx -o json | jq '.metadata.name'

# Get nested field
kubectl get pod nginx -o json | jq '.spec.containers[0].image'

# Remove quotes from output
kubectl get pod nginx -o json | jq -r '.metadata.name'
```

### Arrays

```bash
# First element
jq '.[0]'

# All elements
jq '.[]'

# Specific index
jq '.[2]'

# Array length
jq '. | length'

# Get field from each array element
kubectl get pods -o json | jq '.items[].metadata.name'
```

### Filtering

```bash
# Select by condition
kubectl get pods -o json | jq '.items[] | select(.status.phase != "Running") | .metadata.name'

# Select by label
kubectl get pods -o json | jq '.items[] | select(.metadata.labels.app == "nginx") | .metadata.name'

# Multiple fields
kubectl get pods -o json | jq '.items[] | {name: .metadata.name, status: .status.phase}'
```

### Common jq Options

| Flag | Meaning |
|------|---------|
| `-r` | Raw output (no quotes around strings) |
| `-c` | Compact output (one line) |
| `-e` | Exit with error if output is null/false |
| `--arg name val` | Pass variable: `jq --arg n "nginx" '.items[] \| select(.metadata.name == $n)'` |

## ⚡ CKA Speed Combos

### Extract Pod Info
```bash
# All pod names
kubectl get pods -o json | jq -r '.items[].metadata.name'

# Pod names + status
kubectl get pods -o json | jq -r '.items[] | "\(.metadata.name) \(.status.phase)"'

# Non-running pods
kubectl get pods -o json | jq -r '.items[] | select(.status.phase != "Running") | .metadata.name'
```

### Extract Node Info
```bash
# Node InternalIP addresses
kubectl get nodes -o json | jq -r '.items[].status.addresses[] | select(.type=="InternalIP") | .address'

# Node names + conditions
kubectl get nodes -o json | jq -r '.items[] | {name: .metadata.name, ready: (.status.conditions[] | select(.type=="Ready") | .status)}'
```

### Extract Container Images
```bash
# All images in use across all pods
kubectl get pods -A -o json | jq -r '.items[].spec.containers[].image' | sort -u
```

### Parse Events
```bash
# Warning events
kubectl get events -o json | jq -r '.items[] | select(.type=="Warning") | "\(.reason): \(.message)"'
```

## 🔀 jq vs jsonpath

| Feature | jq | jsonpath |
|---------|-------|---------|
| Syntax | `jq -r '.items[].metadata.name'` | `-o jsonpath='{.items[*].metadata.name}'` |
| Needs install? | Usually pre-installed | Built into kubectl |
| Power | Very powerful filtering | Basic filtering |
| **CKA recommendation** | Use for complex queries | Use for simple queries |

For CKA: **jsonpath is usually enough**. Use jq when you need `select()` filtering.

## 🎯 CKA Exam Patterns

| Scenario | Command |
|----------|---------|
| Get pod names | `kubectl get pods -o json \| jq -r '.items[].metadata.name'` |
| Find non-running pods | `... \| jq '.items[] \| select(.status.phase != "Running")'` |
| Get container images | `... \| jq -r '.items[].spec.containers[].image'` |
| Get node IPs | `... \| jq -r '.items[].status.addresses[] \| select(.type=="InternalIP") \| .address'` |
| Custom output | `... \| jq -r '"\(.metadata.name) \(.status.phase)"'` |

---

## 💡 Pro Tips

1. **`-r` = raw** — always use when you want plain text (no JSON quotes)
2. **jsonpath for simple, jq for complex** — don't overcomplicate simple queries
3. **`select()` is jq's killer feature** — filter by any condition
4. **String interpolation**: `"\(.field1) \(.field2)"` — format custom output
5. **`| length`** — quick count of array elements
6. **Pipe within jq**: `jq '.items[] | .metadata | .name'` — chain filters
